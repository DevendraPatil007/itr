17. JAVASCRIPT
TOPIC :- Prototype in JavaScript

Every JavaScript FUNCTION (not regular object) has a built-in
property called prototype, which is an object used to share
methods and properties across all instances created from that function.

---

What is Prototype?

Only FUNCTIONS have a prototype property by default.
Regular objects do NOT have prototype (it is undefined for them).

function hello(){}
console.log(hello.prototype);   // {} (empty object)

const obj = { name: "Yatin" };
console.log(obj.prototype);     // undefined

Checking prototype existence:-

if (printfunction.prototype) {
    console.log("prototype is present.");
} else {
    console.log("prototype is not present.");
}

---

Adding to prototype

You can attach properties and methods directly onto a function's prototype.
These become available to every instance created from that function.

function hello(){
    console.log("hello world");
}

hello.prototype.abcproperty = "abc";
hello.prototype.xyzproperty = "xyz";
hello.prototype.sing = () => {
    console.log("lalalala");
};

hello.prototype.sing();          // "lalalala"
console.log(hello.prototype.xyzproperty);  // "xyz"

---

Why Prototype Matters (Inheritance)

All JavaScript objects inherit properties and methods from a prototype:-

   Array objects   inherit from Array.prototype
   Date objects    inherit from Date.prototype
   Custom objects  inherit from their Constructor.prototype

Object.prototype sits at the TOP of the chain — everything
eventually inherits from it.

Diagram:-

Person instance
      │
      ▼  (inherits from)
Person.prototype
      │
      ▼  (inherits from)
Object.prototype
      │
      ▼
     null

---

Key Points

✅ Only functions have a prototype property by default (objects don't)
✅ prototype is an empty object {} until you add things to it
✅ Methods added to prototype are shared across ALL instances
✅ Every type in JS (Array, Date, custom constructors) has its own prototype
✅ Object.prototype is the top of the inheritance chain

JAVASCRIPT
TOPIC :- The new Keyword

The new keyword creates an instance of an object using a constructor function.

---

What happens when you use `new`?

When you call a function with `new`, 4 things happen automatically:-

1. A new EMPTY object is created          ->  this = {}
2. The new object's [[Prototype]] (__proto__)
   is set to the constructor function's prototype
3. `this` is bound to point to the newly created object
4. The object is returned automatically (unless you return
   something else explicitly)

---

Code used :-

JavaScript

function createUser(first_name, age) {
    this.first_name = first_name;
    this.age = age;
}

createUser.prototype.about = function () {
    console.log(this.first_name, this.age);
};

const user1 = new createUser("Yatin", 22);
console.log(user1);                  // { first_name: "Yatin", age: 22 }
console.log(createUser.prototype);   // { about: f }
user1.about();                       // "Yatin 22"

---

Behind the scenes (manual simulation)

What `new createUser("Yatin", 22)` essentially does:-

   this = {}                                  // step 1
   this.__proto__ = createUser.prototype       // step 2
   createUser.call(this, "Yatin", 22)          // step 3 (binds this)
   return this                                 // step 4

This is also why you could manually achieve the same using:-

   const user = Object.create(createUser.prototype);
   // no need to apply `this` manually for proto linking

---

Key Points

✅ new automatically creates {}, links prototype, binds this, and returns object
✅ Constructor functions are named with a Capital letter by convention
✅ Without `new`, `this` inside the function refers to the global object (or undefined in strict mode)
✅ Object.create(Fn.prototype) is the manual equivalent of step 1+2 of `new`
✅ Methods should go on .prototype, not inside the constructor (saves memory)

 JAVASCRIPT
TOPIC :- Constructor Functions & Prototype Methods

Constructor functions are regular functions used as a blueprint
to create multiple similar objects, with shared methods stored on prototype.

---

Why use prototype for methods?

If methods are defined INSIDE the constructor function,
a NEW COPY of that method is created for EVERY instance — wasting memory.

If methods are defined on the PROTOTYPE,
all instances SHARE the same method — only one copy exists in memory.

---

Code used :-

JavaScript

function CreateUser(first_name, last_name, email, age, address) {
    this.first_name = first_name;
    this.last_name = last_name;
    this.email = email;
    this.age = age;
    this.address = address;
    return this;
}

// Methods added to prototype (shared across all instances)
CreateUser.prototype.about = function () {
    return `${this.first_name} ${this.last_name} is ${this.age} years old.`;
};

CreateUser.prototype.is18 = function () {
    return this.age >= 18;
};

CreateUser.prototype.sing = function () {
    return 'abc efg hijk lmno lmno pq rs t u v w';
};

const user1 = new CreateUser("Yatin", "chaudhari", "c.yatin727@gmail.com", 25, "home,building,pincode,state");

console.log(user1);
console.log(user1.is18());    // true
console.log(user1.about());   // "Yatin chaudhari is 25 years old."

---

Key Points

✅ Constructor functions start with a Capital letter by convention
✅ Define shared methods on .prototype, not inside the function body
✅ Every instance created with `new` automatically gets access to prototype methods
✅ Properties (first_name, age, etc.) are unique per instance — set inside constructor
✅ Methods (about, is18, sing) are shared — set on prototype

 JAVASCRIPT
TOPIC :- Object.create() — Manual Prototype Linking

Object.create() creates a new object and manually links its
prototype, without using the `new` keyword.

---

Why use Object.create()?

It gives full manual control over prototype linking —
useful for understanding what `new` does internally,
or for prototypal inheritance patterns without constructors.

---

Code used :-

JavaScript

function createUser(first_name, last_name, email, age, address) {
    const user = Object.create(createUser.prototype);
    user.first_name = first_name;
    user.last_name = last_name;
    user.email = email;
    user.age = age;
    user.address = address;
    return user;     // must return manually (no `new` magic here)
}

createUser.prototype.about = function () {
    return `${this.first_name} ${this.last_name} is ${this.age} years old.`;
};

createUser.prototype.is18 = function () {
    return this.age >= 18;
};

const user1 = createUser("Yatin", "chaudhari", "c.yatin727@gmail.com", 25, "home,building,pincode,state");

console.log(user1);
console.log(user1.is18());     // true
console.log(user1.about());    // "Yatin chaudhari is 25 years old."

---

new keyword vs Object.create()

Step                          | new keyword        | Object.create()
-------------------------------|--------------------|-----------------------
Creates empty object           | automatic          | manual (Object.create)
Links __proto__ to prototype   | automatic           | manual (via Object.create)
Binds `this`                   | automatic           | manual (assign properties)
Returns object                 | automatic           | must `return` explicitly
Called with `new`               | yes                | no

---

Key Points

✅ Object.create(Fn.prototype) manually does what `new` automates internally
✅ Without `new`, you must explicitly create and return the object yourself
✅ __proto__ is the actual reference; prototype is the object holding shared methods
✅ This pattern is called the Prototypal/Factory pattern
✅ Useful for understanding what happens "under the hood" of `new`

 JAVASCRIPT
TOPIC :- hasOwnProperty() & for...in Loop

hasOwnProperty() checks if a property belongs DIRECTLY to an object,
rather than being inherited from its prototype chain.

---

The Problem with for...in

A for...in loop iterates over ALL enumerable properties of an object,
INCLUDING those inherited from the prototype chain.

This can cause unwanted prototype methods/properties to show up
when you only want the object's OWN data.

---

What is hasOwnProperty()?

Syntax:-
   object.hasOwnProperty(propertyName)

Returns:-
   true  -> property exists directly ON the object
   false -> property is inherited (from prototype) or doesn't exist

---

Code used :-

JavaScript

function CreateUser(first_name, last_name, email, age, address) {
    this.first_name = first_name;
    this.last_name = last_name;
    this.email = email;
    this.age = age;
    this.address = address;
    return this;
}

CreateUser.prototype.about = function () {
    return `${this.first_name} ${this.last_name} is ${this.age} years old.`;
};

const user1 = new CreateUser("Yatin", "chaudhari", "c.yatin727@gmail.com", 25, "home,building,pincode,state");

// Without hasOwnProperty check (would also loop prototype methods if enumerable)
for (let key in user1) {
    console.log(key);
}

// With hasOwnProperty check (only own properties)
for (let key in user1) {
    if (user1.hasOwnProperty(key)) {
        console.log(key);    // first_name, last_name, email, age, address
    }
}

---

Output Comparison

Without check  -> first_name, last_name, email, age, address, about(if enumerable)
With check      -> first_name, last_name, email, age, address  (only own data)

---

Key Points

✅ hasOwnProperty() returns true only for the object's OWN properties
✅ Prevents inherited prototype properties from leaking into for...in loops
✅ Always use hasOwnProperty() inside for...in when looping over object data
✅ Returns a boolean value
✅ Modern alternative: Object.keys(obj) only returns own enumerable properties (no need for hasOwnProperty)

 JAVASCRIPT
TOPIC :- More About Prototype — Built-in Types

All built-in JavaScript types (Array, Date, Object, etc.)
have their own prototype object that provides their methods.

---

Where do array methods come from?

When you call numbers.push() or numbers.map(),
these methods don't exist directly ON the array —
they come from Array.prototype.

---

Code used :-

JavaScript

let numbers = [1, 2, 3, 4, 5, 6];
let arr = new Array(2, 3, 4, 65, 7);   // Array constructor (same as [])

console.log(numbers);
console.log(arr);

console.log(Array.prototype);                    // shows all array methods
console.log(Object.getPrototypeOf(numbers));      // same as Array.prototype

---

Prototype Inheritance Chain (built-in types)

All JavaScript objects inherit properties and methods from a prototype:-

   Date objects    inherit from Date.prototype
   Array objects    inherit from Array.prototype
   Custom objects   inherit from CustomConstructor.prototype

Object.prototype sits at the TOP of every chain:-

   Array.prototype, Date.prototype, Person.prototype
   all inherit from Object.prototype

---

Adding methods to built-in prototypes (use with caution)

You CAN add new methods to all arrays at once by modifying Array.prototype,
but this is generally discouraged in real projects (can break other code/libraries).

Array.prototype.myCustomMethod = function () {
    // now ALL arrays have this method
};

---

Key Points

✅ Object.getPrototypeOf(obj) returns the prototype an object inherits from
✅ [] and new Array() both use Array.prototype internally
✅ Built-in objects (Array, Date, Object) each expose their methods via .prototype
✅ Object.prototype is the root of the entire prototype chain
✅ Modifying built-in prototypes directly is possible but considered bad practice
