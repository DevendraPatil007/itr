## 1. SYNCHRONOUS vs ASYNCHRONOUS PROGRAMMING

### 1.1 Synchronous Programming

JavaScript is a **Synchronous, Single-Threaded** language by default.

> Operations execute **sequentially** — the next line only runs after the
> current one finishes.

```js
console.log("script Start");

for (let i = 1; i < 10000; i++) {
    console.log("inside for Loop");   // blocks until done
}

console.log("script end");
// Output order: "script Start" → all loop logs → "script end"
```

```
SYNCHRONOUS EXECUTION MODEL
┌─────────────────────────────────────────────────┐
│              CALL STACK (Single Thread)         │
│                                                 │
│  Step 1 → console.log("script Start")          │
│  Step 2 → for loop runs completely (blocking)  │
│  Step 3 → console.log("script end")            │
│                                                 │
│  Each step WAITS for previous to finish ✅      │
└─────────────────────────────────────────────────┘
```

✅ Simple and predictable
❌ Slow operations (network, timers) block everything

## 2. setTimeout() METHOD

> Executes a function **once** after a specified delay (in milliseconds).

### 2.1 Basic Syntax

```js
// Named function
function hello() {
    console.log("hello world!");
}
setTimeout(hello, 5000);   // runs after 5 seconds

// Arrow function (inline)
setTimeout(() => {
    console.log("inside Timeout Function");
}, 3000);
```

### 2.2 How It Works (Event Loop)

```
ASYNCHRONOUS EXECUTION — setTimeout FLOW
                                                        
  JS Engine (Single Thread)                            
  ┌────────────────────────┐                           
  │  console.log("Start")  │ ← executes immediately   
  │  setTimeout(fn, 3000)  │ ← handed off to Web API  
  │  console.log("End")    │ ← executes immediately   
  └────────────────────────┘                           
           │                                           
           ▼ handed off                               
  ┌────────────────────────┐                           
  │       WEB APIs         │                           
  │  Timer: wait 3000ms    │                           
  └────────────┬───────────┘                           
               │ after delay                          
               ▼                                      
  ┌────────────────────────┐                           
  │     CALLBACK QUEUE     │                           
  │   [ fn() is waiting ]  │                           
  └────────────┬───────────┘                           
               │ when call stack is EMPTY             
               ▼                                      
  ┌────────────────────────┐                           
  │      EVENT LOOP        │ ← moves fn → call stack  
  └────────────────────────┘                           
```

> 🔑 **Event Loop:** Pulls tasks from the Callback Queue and pushes them onto
> the Call Stack only when the Call Stack is **empty**.

### 2.3 setTimeout with delay = 0 (tricky!)

```js
console.log("start Script");

const id = setTimeout(() => {
    console.log("inside Timeout Function");  // still runs LAST
}, 0);

for (let i = 1; i < 10000; i++) {
    console.log("....");   // for loop runs first, blocks stack
}

console.log("settime out id is ", id);
console.log("clearing time out ");
clearTimeout(id);           // cancels the timeout before it runs
console.log("Script Ends");
```

```
Output order:
  1. "start Script"
  2. "...." × 9999  (for loop blocks the stack)
  3. "settime out id is 1"
  4. "clearing time out"
  5. "Script Ends"
  6. setTimeout callback → NEVER runs (clearTimeout cancelled it)
```

✅ `clearTimeout(id)` — cancels a pending setTimeout using its returned ID.

## 3. setInterval() METHOD

> Executes a function **repeatedly** at every given time interval, until stopped.

### 3.1 Basic Syntax

```js
console.log("start Script");

setInterval(() => {
    console.log(Math.random());   // prints a new random number every 1 second
}, 1000);

console.log("Script Ends");
```

### 3.2 Practical Example — Random Background Color Changer

```js
// set_interval_prac.js

const body = document.body;

const intervalID = setInterval(() => {
    const red   = Math.floor(Math.random() * 256);
    const blue  = Math.floor(Math.random() * 256);
    const green = Math.floor(Math.random() * 256);
    const rgb   = `rgb(${red},${green},${blue})`;
    console.log(rgb);
    body.style.backgroundColor = rgb;
}, 1000);

const stopButton = document.querySelector("button");

stopButton.addEventListener("click", (e) => {
    e.preventDefault();
    clearTimeout(intervalID);                            // stops the interval
    stopButton.textContent = body.style.backgroundColor; // shows last color
});
```

```
setInterval FLOW
┌─────────────────────────────────────────────┐
│  setInterval(fn, 1000)                      │
│       │                                     │
│       ▼                                     │
│  [fn runs] → wait 1s → [fn runs] → wait 1s │
│       └─── repeats until clearInterval() ──┘│
└─────────────────────────────────────────────┘
```

✅ `clearInterval(id)` / `clearTimeout(id)` — both work to stop an interval.

## 4. CALLBACK FUNCTIONS

### 4.1 What is a Callback?

> A **callback** is a function passed as an **argument** to another function,
> to be executed after that function completes its task.

```js
function myfunc(callback) {
    console.log("function is doing task 1");
    callback();                             // calls the passed function
}

function myfunc2() {
    console.log("function is doing task 2");
}

myfunc(myfunc2);
// Output:
// "function is doing task 1"
// "function is doing task 2"
```

### 4.2 Anonymous & Arrow Function as Callbacks

```js
// Anonymous function expression as callback
myfunc(function () {
    console.log("function is doing task 2");
});

// Arrow function as callback
myfunc(() => {
    console.log("function is doing task 3");
});
```

### 4.3 Callbacks with Arguments

```js
function getTwonumbersandAdd(num1, num2, callback) {
    console.log(num1, num2);
    callback(num1, num2);
}

function addTwonumbers(number1, number2) {
    console.log(number1 + number2);
}

getTwonumbersandAdd(4, 5, addTwonumbers);   // Output: 9
```

### 4.4 Callbacks with Validation (onSuccess / onFailure)

```js
function getTwonumbersandAdd(num1, num2, onSuccess, onFailure) {
    if (typeof num1 === "number" && typeof num2 === "number") {
        onSuccess(num1, num2);
    } else {
        onFailure();
    }
}

getTwonumbersandAdd(
    4, "5",
    (number1, number2) => {
        console.log(number1 + number2);   // success path
    },
    () => {
        alert("Invalid Output!!");        // failure path
        console.log("Enter Valid Number");
    }
);
```

```
CALLBACK PATTERN — FLOW DIAGRAM
                                                  
  getTwonumbersandAdd(4, "5", onSuccess, onFailure)
              │                                   
              ▼                                   
     typeof check valid?                          
        /           \                             
      YES             NO                          
       │               │                          
       ▼               ▼                          
  onSuccess()     onFailure()                     
  console.log()   alert() + console.log()         
```

## 5. CALLBACK HELL / PYRAMID OF DOOM

### 5.1 What is Callback Hell?

> **Callback Hell** = Multiple nested callbacks stacked inside each other,
> forming a deep pyramid shape. Each callback waits for the previous one.

```js
// Without callback hell — independent, no order guarantee
setTimeout(() => { heading1.textContent = "Heading 1"; }, 1000);
setTimeout(() => { heading2.textContent = "Heading 2"; }, 3000);
// ↑ These run independently, order NOT guaranteed
```

### 5.2 The Pyramid of Doom (Full Example)

```js
// changeText(element, text, color, time, onSuccess, onFailure)

changeText(heading1, "one", "violet", 3000, () => {
    changeText(heading2, "two", "purple", 2000, () => {
        changeText(heading3, "three", "red", 1000, () => {
            changeText(heading4, "four", "pink", 1000, () => {
                changeText(heading5, "five", "green", 2000, () => {
                    changeText(heading6, "six", "blue", 2000, () => {
                        changeText(heading7, "seven", "cyan", 2000, () => {
                            changeText(heading8, "eight", "#cda562", 1000, () => {
                                changeText(heading9, "nine", "#dca652", 1000, () => {
                                    changeText(heading10, "ten", "brown", 1000,
                                        () => { /* done */ },
                                        () => { console.log("Error: heading10"); }
                                    );
                                }, () => { console.log("Error: heading9"); });
                            }, () => { console.log("Error: heading8"); });
                        }, () => { console.log("Error: heading7"); });
                    }, () => { console.log("Error: heading6"); });
                }, () => { console.log("Error: heading5"); });
            }, () => { console.log("Error: heading4"); });
        }, () => { console.log("Error: heading3"); });
    }, () => { console.log("Error: heading2"); });
}, () => { console.log("Error: heading1"); });
```

```
PYRAMID OF DOOM — VISUAL SHAPE
                                                          
  changeText(h1, ..., () => {              ← level 1    
      changeText(h2, ..., () => {          ← level 2    
          changeText(h3, ..., () => {      ← level 3    
              changeText(h4, ..., () => {  ← level 4    
                  ...                                    
              });                                        
          });                                            
      });                                               
  });                                                   
  
  Code grows RIGHTWARD → hard to read, debug, maintain
```

### 5.3 The changeText Helper Function

```js
function changeText(element, text, color, time, onSuccess, onFailure) {
    setTimeout(() => {
        if (element) {
            element.textContent = text;
            element.style.color = color;
            if (onSuccess) onSuccess();     // chain next step
        } else {
            if (onFailure) onFailure();     // handle missing element
        }
    }, time);
}
```

## 6. QUICK REFERENCE — KEY POINTS

```
✅ JS is synchronous + single-threaded by default
✅ setTimeout(fn, ms)   → runs fn ONCE after ms milliseconds
✅ setInterval(fn, ms)  → runs fn EVERY ms milliseconds (repeats)
✅ clearTimeout(id)     → cancels a pending setTimeout
✅ clearInterval(id)    → stops a running setInterval
✅ setTimeout(fn, 0)    → still runs AFTER current call stack clears
✅ Callback = function passed as argument, called later
✅ Anonymous function   = function without a name
✅ Callback Hell        = deeply nested callbacks (pyramid shape)
✅ Event Loop           = moves tasks from queue → call stack when empty
✅ Web APIs             = handle timers/fetch outside the JS thread
```

## 2. XMLHttpRequest (XHR) — OLD WAY

### 2.1 XHR readyState Phases

```
XHR readyState VALUES
┌──────────┬───────────────────────────────────────────────────┐
│  Value   │  Meaning                                          │
├──────────┼───────────────────────────────────────────────────┤
│    0     │  UNSENT — xhr created, open() not called yet      │
│    1     │  OPENED — open() called                           │
│    2     │  HEADERS_RECEIVED — send() called, headers got    │
│    3     │  LOADING — response body downloading              │
│    4     │  DONE — response fully received ✅                │
└──────────┴───────────────────────────────────────────────────┘
Ref: https://developer.mozilla.org/en-US/docs/Web/API/XMLHttpRequest/readyState
```

### 2.2 Basic XHR GET Request

```js
const url = "https://jsonplaceholder.typicode.com/posts";

const xhr = new XMLHttpRequest();   // step 1: create XHR object

xhr.open("GET", url);               // step 2: configure method + URL

// onreadystatechange fires every time readyState changes
xhr.onreadystatechange = function () {
    if (xhr.readyState === 4) {
        const response = xhr.response;         // JSON string
        const data = JSON.parse(response);     // convert to JS object
        console.log(data);
    }
};

xhr.send();                         // step 3: send the request
```

### 2.3 Using onload (Cleaner — fires only at readyState 4)

```js
xhr.onload = function () {
    const data = JSON.parse(xhr.response);
    console.log(data);
};

xhr.send();
```

### 2.4 XHR Flow Diagram

```
XHR REQUEST LIFECYCLE
                                                          
  new XMLHttpRequest()                                   
          │                                              
          ▼                                              
  xhr.open("GET", url)    ← readyState: 0 → 1           
          │                                              
          ▼                                              
  xhr.send()              ← readyState: 1 → 2 → 3 → 4  
          │                                              
          ▼                                              
  xhr.onreadystatechange / xhr.onload fires              
          │                                              
          ├── readyState === 4 && status 200-299?        
          │        │                                     
          │      YES → JSON.parse(xhr.response) → use data
          │        │                                     
          │       NO → handle error                      
          │                                              
  xhr.onerror → fires on network failure                 
```

## 3. XHR ERROR HANDLING

### 3.1 Status Code Based Error Handling

```js
const URL = "https://jsonplaceholder.typicode.com/posts/2";
const xhr = new XMLHttpRequest();

xhr.open("GET", URL);

xhr.onload = () => {
    if (xhr.status >= 200 && xhr.status < 300) {
        const data = JSON.parse(xhr.response);
        console.log(data);   // ✅ success
    } else {
        console.log("something went wrong!!");   // ❌ HTTP error
    }
};

// Fires on network failure (no internet, server down)
xhr.onerror = () => {
    console.log("network error!");
};

xhr.send();
```

### 3.2 Chained XHR Requests (Callback Style)

```js
// Fetch all posts → get id of 4th → fetch that specific post
const URL = "https://jsonplaceholder.typicode.com/posts";
const xhr = new XMLHttpRequest();

xhr.open("GET", URL);

xhr.onload = () => {
    if (xhr.status >= 200 && xhr.status < 300) {
        const data = JSON.parse(xhr.response);
        const id = data[3].id;                          // get 4th post id

        const xhr2 = new XMLHttpRequest();
        const URL2 = `${URL}/${id}`;                    // dynamic URL

        xhr2.open("GET", URL2);
        xhr2.onload = () => {
            const data2 = JSON.parse(xhr2.response);
            console.log(data2);
        };
        xhr2.send();
    }
};

xhr.onerror = () => { console.log("network error!"); };
xhr.send();
```

```
HTTP STATUS CODE RANGES
┌──────────────┬───────────────────────────────────────┐
│  Range       │  Meaning                              │
├──────────────┼───────────────────────────────────────┤
│  200 – 299   │  ✅ Success                           │
│  300 – 399   │  🔀 Redirect                          │
│  400 – 499   │  ❌ Client Error (404 = Not Found)    │
│  500 – 599   │  💥 Server Error                      │
└──────────────┴───────────────────────────────────────┘
Ref: https://developer.mozilla.org/en-US/docs/Web/HTTP/Status
```

## 4. XHR USING PROMISES

> Wrap XHR inside a Promise to avoid nested callbacks.

### 4.1 Creating the Promise Wrapper

```js
const URL = "https://jsonplaceholder.typicode.com/posts";

function sendRequest(method, url) {
    return new Promise(function (resolve, reject) {

        const xhr = new XMLHttpRequest();
        xhr.open(method, url);

        xhr.onload = function () {
            if (xhr.status >= 200 && xhr.status < 300) {
                resolve(xhr.response);          // ✅ fulfill promise
            } else {
                reject(new Error("Something Went Wrong!!"));  // ❌ reject
            }
        };

        xhr.onerror = function () {
            reject(new Error("Server Side Error or Network Down"));
        };

        xhr.send();
    });
}
```

### 4.2 Consuming the Promise (Chained .then())

```js
sendRequest("GET", URL)
    .then((response) => {
        const data = JSON.parse(response);
        return data;                        // pass to next .then()
    })
    .then(data => {
        console.log(data);
        const id = data[4].id;
        return id;
    })
    .then(id => {
        const url = `${URL}/${id}`;
        return sendRequest("GET", url);     // return another promise
    })
    .then(newResponse => {
        const newData = JSON.parse(newResponse);
        console.log(newData);
    })
    .catch(error => {
        console.log(error);                 // catches any error above
    });
```

```
PROMISE CHAIN FLOW
                                                    
  sendRequest("GET", URL)                          
          │                                        
          ▼                                        
    .then(parse + return data)                     
          │                                        
          ▼                                        
    .then(extract id + return id)                  
          │                                        
          ▼                                        
    .then(build URL + return sendRequest(...))     
          │                                        
          ▼                                        
    .then(parse + log final data)                  
          │                                        
          ▼                                        
    .catch(handle any error in chain) ◄────────────
                                   (catches all above)
```

## 5. FETCH API — MODERN WAY

> Built-in browser API, returns a **Promise** automatically.
> Much cleaner than XHR.

### 5.1 Basic GET with fetch

```js
const URL = "https://jsonplaceholder.typicode.com/posts";

const promise = fetch(URL);   // returns a Promise

promise
    .then(response => {
        console.log(response);          // Response object (not data yet!)
        return response.json();         // .json() also returns a Promise
    })
    .then(data => {
        console.log(data);              // actual JS array/object
    })
    .catch(error => {
        console.log(error);
    });
```

### 5.2 ⚠️ Fetch 404 Gotcha — Manual Error Handling Required

```
IMPORTANT:
fetch() does NOT reject on HTTP errors (404, 500 etc.)
It only rejects on network failure (no internet, DNS fail).
→ You must check response.ok manually!
```

```js
fetch(URL)
    .then((response) => {
        if (response.ok) {                          // status 200-299
            return response.json();
        } else {
            throw new Error("Something Went Wrong!!");  // manual throw
        }
    })
    .then((data) => {
        console.log(data);
    })
    .catch((error) => {
        console.error(error);   // catches network errors + thrown errors
    });
```

### 5.3 POST Request with fetch

```js
const URL = "https://jsonplaceholder.typicode.com/posts";

fetch(URL, {
    method: 'POST',
    body: JSON.stringify({           // convert JS object → JSON string
        title: 'foo',
        body: 'bar',
        userId: 1,
    }),
    headers: {
        'Content-type': 'application/json; charset=UTF-8',
    },
})
    .then((response) => {
        if (response.ok) {
            return response.json();
        } else {
            throw new Error("Something Went Wrong!!");
        }
    })
    .then((data) => {
        console.log(data);           // { id: 101, title: 'foo', ... }
    })
    .catch((error) => {
        console.error(error);
    });
```

```
FETCH API FLOW
                                                       
  fetch(URL)              ← returns Promise<Response> 
          │                                            
          ▼                                            
  .then(response =>)                                  
      response.ok? ──── NO ──► throw new Error()      
          │                          │                 
         YES                         │                 
          │                          │                 
  return response.json() ◄───────────┘ (caught below) 
  (returns Promise<data>)                              
          │                                            
          ▼                                            
  .then(data => use it)                               
          │                                            
          ▼                                            
  .catch(error => handle)                             
```

## 6. ASYNC / AWAIT

> **Syntactic sugar** over Promises — makes async code look synchronous.
> `async` function **always returns a Promise**.
> `await` pauses execution inside the function until the Promise resolves.

### 6.1 Syntax

```js
// async function declaration
async function getPosts() {
    const response = await fetch(URL);   // waits for fetch to resolve
    if (!response.ok) {
        throw new Error("Something went Wrong!");
    }
    const data = await response.json();  // waits for .json() to resolve
    return data;                         // returned as resolved Promise
}

// async arrow function (equivalent)
const getPosts = async () => {
    const response = await fetch(URL);
    if (!response.ok) {
        throw new Error("Something went Wrong!");
    }
    const data = await response.json();
    return data;
};
```

### 6.2 Consuming async Function

```js
console.log("Script Start");

const URL = "https://jsonplaceholder.typicode.com/posts";

const getPosts = async () => {
    const response = await fetch(URL);
    if (!response.ok) {
        throw new Error("Something went Wrong!");
    }
    const data = await response.json();
    return data;
};

// async function returns a Promise → consume with .then/.catch
getPosts()
    .then(data => { console.log(data); })
    .catch(error => { console.error(error); });

console.log("Script End");
```

```
OUTPUT ORDER:
  1. "Script Start"   ← synchronous
  2. "Script End"     ← synchronous (getPosts runs async in background)
  3. [array of posts] ← after fetch resolves
```

### 6.3 Async/Await Flow Diagram

```
ASYNC / AWAIT EXECUTION
                                                            
  console.log("Script Start")   ← runs immediately        
          │                                                 
          ▼                                                 
  getPosts() called                                        
  ┌─────────────────────────────────────┐                  
  │  async function body                │                  
  │  const response = await fetch(URL)  │ ← pauses HERE   
  │  ... waits for fetch ...            │   (non-blocking) 
  │  const data = await response.json() │ ← pauses again  
  │  return data                        │                  
  └─────────────────────────────────────┘                  
          │                                                 
          ▼ (while waiting above)                          
  console.log("Script End")     ← runs immediately        
          │                                                 
          ▼ (when fetch resolves)                          
  .then(data => console.log(data))                        
```

## 8. ES6 MODULES (Brief Note)

> **ES6 Modules** allow splitting JS code into separate files using
> `import` and `export` keywords.

```js
// math.js — named export
export function add(a, b) { return a + b; }
export const PI = 3.14159;

// Default export
export default function multiply(a, b) { return a * b; }

// main.js — import
import multiply, { add, PI } from './math.js';

console.log(add(2, 3));      // 5
console.log(PI);             // 3.14159
console.log(multiply(4, 5)); // 20
```

```html
<!-- In HTML — use type="module" to enable ES6 modules -->
<script type="module" src="main.js"></script>
```
